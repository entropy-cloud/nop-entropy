# Nop平台AI代理命名约定文档 (上古创世神系)

## 1. 概述

为统一Nop平台AI代理的命名体系，增强文化内涵与技术品牌辨识度，特制定本命名约定。本方案采用**上古创世神系**为命名基准，构建层次清晰、寓意深远的AI代理生态系统。

## 2. 命名原则

### 2.1 核心原则
- **文化一致性**：所有名称均源自中国上古神话传说
- **职责对应性**：神祇职能与代理技术职责高度匹配
- **层次结构性**：反映代理在系统架构中的层级关系
- **扩展预留性**：为未来新增代理类型预留命名空间

### 2.2 命名格式
```
中文神祇名 (英文拼音) [英文角色代号]
```
**示例**：
```
盘古 (Pangu) [Core-Creator]
后稷 (Houji) [Domain-Expert]
```

### 4.2 代理工作流命名
基于Delta Pipeline的协作流程命名：

```
{迭代阶段}-{主神名}-{功能领域}
```

**示例**：
- `初开-盘古-核心框架`
- `建模-仓颉-用户实体`
- `疏流-大禹-订单流程`
- `铸器-鲁班-报表批处理`
- `织梦-瑶姬-管理后台`

## 6. 扩展与演化规则

### 6.1 新增代理命名流程
1. **确定职责范畴**：明确新代理的技术职责
2. **寻找神祇对应**：在上古神话中寻找职能匹配的神祇
3. **检查名称冲突**：确保名称在现有体系中唯一
4. **更新本文档**：将新代理纳入相应层级和表格

### 6.2 代理别名机制
允许在特定上下文中使用别名：
- **技术上下文**：使用英文角色代号（如 `Domain-Expert`）
- **文化上下文**：使用中文神祇名（如 `后稷`）
- **团队内部**：可使用简称（如 `稷神`、`禹工`）

### 6.3 代理升级与降级
- **代理能力扩展**：可在本层级内扩展职责，名称不变
- **代理职责重大变更**：可能需要重新匹配神祇，更名
- **代理停用**：标记为`归隐`状态，名称保留不再复用

## 7. 实施指南

### 7.1 第一阶段：核心代理命名
1. 确定`盘古`、`伏羲`为平台创建者代理
2. 确定`轩辕`为总架构师代理
3. 根据当前需求，从第三层（领域专家）选取3-5个核心代理

### 7.2 第二阶段：扩展团队命名
1. 建立`轩辕之庭`作为默认完整团队
2. 根据项目特点组建专业团队（如`瑶姬画院`、`大禹工坊`）

### 7.3 第三阶段：全生态应用
1. 在配置文件中全面应用本命名体系
2. 在文档、日志、监控中统一使用
3. 培训团队成员理解命名背后的文化内涵
