## 格式要求

### 1. 章节结构

所有文档必须包含以下章节：

1. **概述** (必须) - 简要介绍文档主题
2. **核心功能 / 基本使用** (必须) - 核心概念或基本使用方法
3. **示例代码** (推荐) - 实际代码示例
4. **最佳实践** (推荐) - 最佳实践和建议
5. **常见问题** (推荐) - 常见问题和解答
6. **相关文档** (必须) - 相关文档链接

### 2. 章节标题

- 一级标题 `#` 用于文档标题
- 二级标题 `##` 用于主要章节
- 三级标题 `###` 用于子章节
- 四级标题 `####` 用于更细分的子章节

### 3. 代码示例

- 使用 ```java 包裹Java代码
- 使用 ```xml 包裹XML配置
- 使用 ```javascript 包裹JavaScript代码
- 使用 ```sql 包裹SQL语句
- 代码示例应该简洁、清晰、可运行

### 4. 链接格式

- 使用相对路径链接到其他文档
- 格式：`[文档名称](./相对路径/文档名称.md)`
- 使用有意义的链接文本

### 5. 列表格式

- 使用 `-` 表示无序列表
- 使用 `1.` `2.` `3.` 表示有序列表
- 列表项之间保持一致的对齐

### 6. 表格格式

- 使用标准的Markdown表格格式
- 表头使用 `|` 分隔
- 对齐使用 `:` 在 `|` 内指定（左对齐 `:---`，居中 `:--:`，右对齐 `---:`）

### 7. 文档元信息

在文档末尾添加以下元信息：

```markdown
---

**文档版本**: 1.0
**最后更新**: YYYY-MM-DD
**作者**: 作者姓名
```

### 8. 语言风格

- 使用中文作为主要语言
- 技术术语使用英文原文
- 代码注释使用英文
- 保持简洁、清晰、准确

### 9. 格式化规范

- 代码块使用4空格缩进
- 列表项使用2空格缩进
- 章节之间保持一个空行
- 代码块前后保持一个空行

## 常见错误

### 错误1: 缺少"最佳实践"章节

**问题**: 文档缺少最佳实践章节。

**解决方案**: 添加"## 最佳实践"章节，列出使用此功能的最佳实践。

### 错误2: 缺少"常见问题"章节

**问题**: 文档缺少常见问题章节。

**解决方案**: 添加"## 常见问题"章节，列出常见问题和解答。

### 错误3: 链接路径错误

**问题**: 链接路径不正确。

**解决方案**: 使用相对路径，确保链接可访问。

### 错误4: 代码示例不完整

**问题**: 代码示例不完整或无法运行。

**解决方案**: 提供完整的、可运行的代码示例。
