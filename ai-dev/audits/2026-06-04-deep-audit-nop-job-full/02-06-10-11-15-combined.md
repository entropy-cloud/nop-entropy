## 维度 15：类型安全与泛型使用

### [维度15-01] Store 实现类中 IOrmEntityDao 转换未标注 @SuppressWarnings

- **文件**: `JobScheduleStoreImpl.java:413-423`、`JobFireStoreImpl.java:315-324`、`JobTaskStoreImpl.java:119-121`
- **证据片段**:
  ```java
  private IOrmEntityDao<NopJobSchedule> scheduleDao() {
      return (IOrmEntityDao<NopJobSchedule>) daoProvider.daoFor(NopJobSchedule.class);
  }
  ```

- **严重程度**: P3
- **现状**: 7 个 DAO 获取方法有未检查转换但未添加 @SuppressWarnings。与 BizModel 中同类转换处理方式不一致。
- **风险**: 编译警告，无运行时风险。
- **建议**: 添加 @SuppressWarnings("unchecked") 保持风格一致。
- **信心水平**: 确定
- **误报排除**: Nop 平台 DAO API 返回类型受限，此转换不可避免。
- **复核状态**: 未复核

- RpcJobInvoker/DefaultJobCancelHandler 中的 @SuppressWarnings 使用合理（Map<String,Object> 取值转换）
- 未发现 Object 类型参数/返回值可收窄的情况
- 未发现不必要的强制类型转换（排除平台 DAO API 所需转换）
