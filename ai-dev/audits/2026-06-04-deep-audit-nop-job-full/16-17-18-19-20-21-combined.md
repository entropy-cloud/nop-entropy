## 维度 20：跨模块契约一致性

### [维度20-01] IJobRetryBridge.onFireFailed 返回 String 但实现始终返回 null

- **文件**: `nop-job/nop-job-api/src/main/java/io/nop/job/api/retry/IJobRetryBridge.java:3-5`
- **证据片段**:
  ```java
  public interface IJobRetryBridge {
      String onFireFailed(JobFireFailedEvent event);
  }
  ```
- **严重程度**: P2
- **现状**: 接口定义了 String 返回值（暗示 retry record ID），但两个生产实现（NopRetryJobRetryBridge 和 NoOpJobRetryBridge）始终返回 null。JobCompletionProcessorImpl 中处理非 null retryRecordId 的逻辑是死代码。
- **风险**: retryRecordId 永远不会被通过 retry bridge 设置。接口契约与实现不一致。
- **建议**: 要么让 NopRetryJobRetryBridge 返回实际的 retry record ID，要么将接口返回值改为 void。
- **信心水平**: 确定
- **误报排除**: 已确认两个生产实现都返回 null。测试中有 mock 返回非 null 值。
- **复核状态**: 未复核
