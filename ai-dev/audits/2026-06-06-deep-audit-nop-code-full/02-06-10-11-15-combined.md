## 维度 02：模块职责与文件边界

### [维度02-01] CodeIndexService 超大类——1916行承担6个领域职责

- **文件**: `nop-code/nop-code-service/src/main/java/io/nop/code/service/impl/CodeIndexService.java`
- **证据片段**:
  ```java
  public class CodeIndexService implements ICodeIndexService {
      private final CodeCacheManager cacheManager = new CodeCacheManager();
      private CodeSearchService searchService;
      private CodeGraphService graphService;
      private CodeQueryService queryService;
      // ... 大量 persist*/delete*/resolve* 私有方法
  }
  ```
- **严重程度**: P2
- **现状**: 1916行的"上帝类"，承担索引编排、数据库持久化、文件记录管理、流分析委托、增量索引、路径校验等6个职责。子服务通过 `new` 手动创建而非 IoC 管理。
- **风险**: 修改持久化逻辑时可能意外破坏索引编排逻辑；子服务绕过 IoC 容器；新开发者认知负担高。
- **建议**: 将持久化方法提取为独立的 CodePersistenceService；子服务改为 IoC 管理。
- **信心水平**: 90%
- **误报排除**: 单文件超过合理阈值近4倍，且 new 子服务绕过了 IoC 容器。
- **复核状态**: 未复核

### [维度02-02] SymbolDTO 272行包含业务逻辑方法——DTO 越界

- **文件**: `nop-code/nop-code-service/src/main/java/io/nop/code/service/api/dto/SymbolDTO.java`
- **证据片段**:
  ```java
  @DataBean
  public class SymbolDTO implements Serializable {
      // ... 25个字段 + fromCodeSymbol() 转换方法
      public boolean isAbstractFlag() {
          return (modifiers & CodeSymbol.MODIFIER_ABSTRACT) != 0;
      }
      public boolean isFinalFlag() {
          return (modifiers & CodeSymbol.MODIFIER_FINAL) != 0;
      }
  }
  ```
- **严重程度**: P3
- **现状**: DTO 包含 `fromCodeSymbol()` 转换工厂方法和6个 `isXxxFlag()` 位运算方法，导致 service.api 包反向依赖 core.model 包。
- **风险**: DTO 包含转换逻辑，违反分层原则。
- **建议**: 将 `fromCodeSymbol()` 移到独立 Converter 类中；`isXxxFlag()` 通过 xmeta computed prop 暴露。
- **信心水平**: 75%
- **误报排除**: `isXxxFlag()` 在 DTO 中做位运算确实越界。
- **复核状态**: 未复核

- **证据片段**:
  ```java
  Object parsed = JsonTool.parseNonStrict(extData);
  if (parsed instanceof Map) {
      @SuppressWarnings("unchecked")
      Map<String, Object> map = (Map<String, Object>) parsed;
  ```
- **严重程度**: P3
- **现状**: 全部9处出现在 JSON 动态解析边界，每处都有 instanceof 守卫，是安全的。
- **风险**: 低。可提取泛型辅助方法减少重复。
- **建议**: 在 ExtDataHelper 中添加 `<T> T getField(String extData, String key, Class<T> type)` 泛型方法。
- **信心水平**: 90%
- **误报排除**: JSON 动态边界的必要 unchecked，不是代码质量问题。
- **复核状态**: 未复核
