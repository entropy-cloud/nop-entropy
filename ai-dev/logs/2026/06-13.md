## 分析：用 Nop Task Flow vs Agno 重写 Goal Driver 的 AI 工作流编排

- 新增 `ai-dev/analysis/2026-06-13-goal-driver-replatform-nop-task-flow-vs-agno.md`
- 方法论：逐一映射 Goal Driver 的 8 个核心机制到两个框架的扩展点（基于源码逐行阅读）
- **结论：Nop Task Flow 更容易实现功能等价物**，核心原因是 `TaskStepReturn.RETURN(nextStepName)` 完美映射 Goal Driver 的 `goto` 状态机语义，而 Agno 的 Router 是"函数分发"而非"状态机跳转"
- 次要原因：Nop 的 `ITaskStepRuntime.saveState()` 天然断点恢复（Goal Driver 缺失）、`TimeoutTaskStepWrapper` 现成、`ASYNC_RETURN(CompletableFuture)` 天然异步
- 代价：失去 AI 可读写 JSON 的优势（XDSL XML 冗长）、强耦合 Nop 平台、学习曲线陡
- 提出混合方案：Nop Task Flow 作为底层执行引擎 + JSON→XML 编译器（AI 仍写 JSON）
