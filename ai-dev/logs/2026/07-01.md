# Dev Log: 2026-07-01

## Summary

补强 `docs-for-ai/02-core-guides/service-layer.md`，把两条容易被 AI 忽略的 BizModel 契约从弱措辞/未列出升级为反模式表硬规则。起因：在 nop-app-erp 执行合规整改时，AI 连续两处违反——(1) 给 BizModel 方法误加 `@SingleSession`；(2) 自定义 `I*Biz` 方法签名漏 `IServiceContext context` 末参。根因是文档说了但措辞弱（"通常是 IServiceContext"）、且未进反模式表，凭原代码惯性盖过文档规则（命中 ai-defaults「阅读即理解」警告症状）。

## Changes

### `docs-for-ai/02-core-guides/service-layer.md`

- **反模式表**：
  - 扩展「`@BizMutation @Transactional`」行 → 涵盖「在 BizModel 方法上叠 `@SingleSession`」，说明 BizModel 管道已自动包事务并提供 ORM Session，`@SingleSession` 是 non-BizModel 场景注解，交叉引用 `../03-runbooks/non-bizmodel-orm-access.md`。
  - 新增行「BizModel/`I*Biz` 自定义方法签名缺 `IServiceContext context` 末参」——对齐基类 `ICrudBiz` 契约（全方法末参为 ctx），说明 ctx 承载 `IUserContext`（身份/数据权限）、缓存、事务上下文，跨服务调用（A 域调 B 域 `I*Biz`）必须透传，否则下游丢身份/跳权限/失缓存。
- **「参数与返回值」段**：新增一条——自定义方法以 `IServiceContext context` 为末参，调用其它 `I*Biz` 时透传 `context`。
- **查询段 / 修改段**：把「最后一个参数通常是 `IServiceContext`」强化为「最后一个参数为 `IServiceContext context`（见「参数与返回值」，跨服务调用需透传）」。

## Why

基类 `ICrudBiz`（`nop-persistence/nop-orm/src/main/java/io/nop/orm/biz/ICrudBiz.java`）所有方法末参均为 `IServiceContext context`，这是服务层契约。补强后，AI 写 BizModel 时会被反模式表直接拦截这两类错误，不必依赖"读到弱措辞并自觉遵守"。

## Changes（续）

### `ai-dev/tools/check-ibiz-interfaces.mjs` — 新增 I*Biz 接口合规检查工具

用 **tree-sitter-java（WASM，经 web-tree-sitter）** 解析，检查 `I*Biz` 接口（`*-dao` 模块 biz 包，`extends ICrudBiz`）的每个方法：(1) 必须有 `@BizQuery`/`@BizMutation`/`@BizAction` 注解；(2) 末参必须是 `IServiceContext`。

**选型过程**：先尝试 ast-grep（本目录已有 `@ast-grep/cli` + `rules/`），实测两处卡死——`has` 谓词找不到 modifiers 子树里的注解节点（无论 `stopBy:end`），`inside` 的 pattern 匹配不了 `extends ICrudBiz<泛型>`（tree-sitter 解析为 `generic_type`）。ast-grep 擅长"模式存在性查找"，对"方法必须带某注解"这类**谓词校验**力不从心。改用 tree-sitter-java WASM（免 native 编译，绕开 node 25 的 node-gyp 失败），精确取 method 节点的 annotations/parameters/extends_interfaces。

**验证**：扫 nop-app-erp 279 个 I*Biz 接口命中 82 处真实违规（38 缺注解 + 44 缺 ctx）；扫 nop-entropy 平台 134 个 I*Biz 接口 0 违规（不误报）。注：tree-sitter-java 的 `extends_interfaces`/`modifiers` 节点 fieldName 为 null，必须用节点 `type` 而非 `childForFieldName` 访问。

**配套**：`package.json` 加 `check:ibiz` script + devDependency `tree-sitter-java`（提供 wasm）/`web-tree-sitter`（运行时）；README 加说明。

### `docs-for-ai/02-core-guides/testing.md` — 强化"测试 BizModel 服务方法必须经 IGraphQLEngine"

新增专节：测试里**不要直调** `bizObj.method(args, context)`（裸 `new ServiceContextImpl()`）——多步方法会报 `update-entity-no-current-session`、context 缺身份/权限。正确做法是 `IGraphQLEngine.newRpcContext` + `executeRpc`（引擎管 session/context/管道）。附"常见误判"：报 `no-current-session` 时不要给 BizModel 加 `@SingleSession`（non-BizModel 注解），根因是测试直调缺 session。起因：nop-app-erp 整改中误加 `@SingleSession`，经 `TestErpInvStockMoveGraphQL`（IGraphQLEngine 调 generateMove）验证移除后仍正常，锁定"测试方式"是真因。
