# AI Dev Log - 2026-07-14

### nop-tcc 分布式事务模块逻辑审计与修复

**背景:** 对 nop-tcc 模块做完整逻辑审计，发现若干代码缺陷与状态机健壮性问题，按确认方案修复。

**Changes:**

1. **重试治理落地（`maxRetryCount`）**:
   - `nop-tcc/model/nop-tcc.orm.xml`: `NopTccRecord` 新增 `RETRY_TIMES` 列（propId 16）
   - `ITccRecordStore.fetchExpiredRecords`: 签名新增 `maxRetryCount` 参数；过滤 `retryTimes < maxRetryCount`，抢占时递增 `retryTimes`。达到上限的事务不再被扫描补偿（符合"忽略，不再处理"契约）
   - `TccEngine.checkExpiredTransactions`: 透传 `maxRetryCount`
   - `TccRecordStore`: `default-max-retry-times` 默认值 10000 → 100

2. **状态机守卫（防 confirm/cancel 互相覆盖中间态）**:
   - `TccTransaction.doConfirmAsync`: 仅当未进入 cancel 路径才允许 confirm
   - `TccTransaction.doCancelAsync`: 仅当未进入 confirm 路径才允许 cancel
   - 此前超时兜底与正常 endAsync 并发时，CONFIRMING/CANCELLING 中间态可被互相覆盖，靠乐观锁兜底；现增加显式状态前置守卫

3. **`aggregateConfirmBranchStatus` 修正（`TccRunner`）**:
   - confirm 阶段出现 cancelled 分支说明状态机被破坏，由静默返回 cancelled 改为抛 `ERR_TCC_INVALID_CONFIRM_BRANCH_STATUS` 报错
   - `TccCoreErrors` 新增对应错误码

4. **异常信息保留（`TccTransaction.endAsync`）**:
   - 业务异常存在时，补偿(confirm/cancel)阶段的 err2 现打 ERROR 日志，避免诊断信息丢失（不使用 addSuppressed）

5. **endTime 落地（`TccRecordStore`）**:
   - record/branch 状态更新到 finished 时写入 `endTime`（此前该字段恒为 null）

6. **死代码清理（`TccRecordStore.newBranchRecord`）**:
   - 删除被立即覆盖的 `setBranchNo(parentBranchNo)` 死代码行

7. **`nop-tcc-web` 测试依赖补齐**:
   - pom.xml 补回 `nop-autotest-junit`/`junit-jupiter`/`nop-ooxml-xlsx`（test scope），与 codegen 模板及其它 `-web` 模块对齐，解决 `JunitBaseTestCase` 找不到的编译错误

### PageProvider 异常增强 + 代码生成模板支持 validateAllPages 单元测试

**Changes:**

1. **`nop-web/.../PageProvider.java`**:
   - `getPage()`: 捕获 `NopException` 时追加 `ARG_PAGE_PATH` param；捕获其他 `Exception` 时包装为 `ERR_WEB_PAGE_LOAD_FAIL`（包含 `pagePath` 参数），确保任何加载失败都能定位到具体页面路径
   - `validateAllPages()`: 并行模式（threadCount > 1）下收集所有 `CompletableFuture`，通过 `FutureHelper.syncGet(FutureHelper.waitAll(futures))` 等待全部完成，确保异步异常能被抛出而非静默吞掉

2. **`nop-web/.../WebErrors.java`**:
   - 新增 `ARG_PAGE_PATH = "pagePath"`（避免与通用 `ARG_PATH` 冲突）
   - 新增 `ERR_WEB_PAGE_LOAD_FAIL` 错误码，用于非 `NopException` 的页面加载失败包装

3. **`nop-codegen/.../orm/{appName}-web/.../{moduleClassPrefix}WebPagesTest.java.xgen`**:
   - 新建 xgen 模板，为每个 `xxx-web` 模块生成 `{ModuleClassPrefix}WebPagesTest.java` 单元测试类
   - 测试类注入 `PageProvider`，通过 `testValidateAllPages()` 调用 `validateAllPages()`，确保所有页面模型编译加载无问题

- `nop-kernel/nop-codegen/src/main/resources/_vfs/nop/templates/orm/{!disableWebModule}{appName}-web/src/test/java/{basePackagePath}/web/{moduleClassPrefix}WebPagesTest.java.xgen` (new file)
