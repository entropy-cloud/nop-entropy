## Plan 26: nop-stream Graph Model 与 Checkpoint 端到端集成

- Plan: `ai-dev/plans/26-nop-stream-graph-model-and-checkpoint-integration.md`
- 状态: **completed**（6 个 Phase 全部完成）
- 验证: `./mvnw test -pl nop-stream` BUILD SUCCESS（56 个测试类，0 failures）

### 变更摘要

**Phase 1 - 图模型执行路径（单链管线）**
- 新增 `StreamTaskInvokable`：替代 placeholder Invokable，wire operators + run source + emit MAX_WATERMARK
- 新增 `StreamExecutionEnvironment.executeWithGraphModel()`：Transformation DAG → StreamGraph → JobGraph → Task → TaskExecutor
- 单链约束验证：多链管线（含 keyBy）抛出 IllegalStateException
- 测试: `TestGraphModelExecution` (5 tests)

**Phase 2 - StreamElement 统一分发与 Barrier 传播**
- `CheckpointBarrier extends StreamElement`：barrier 融入流元素类型体系
- `Output.emitBarrier()` + `Input.processBarrier()`：统一分发模型
- `ChainingOutput`/`TimestampedCollector`/`KeyExtractingOutput` 均适配 barrier 转发
- `StreamSourceOperator.injectBarrier()`：Source 端 barrier 注入点
- 测试: `TestBarrierPropagation` (7 tests)

**Phase 3 - 算子状态快照实现**
- 新增 `StateSnapshotContext`：checkpoint 上下文（checkpointId + timestamp）
- `AbstractStreamOperator.snapshotState()`/`restoreState()`：算子级状态快照与恢复
- `IKeyedStateBackend.snapshotState()`/`restoreState()` + `MemoryKeyedStateBackend` 实现
- `StreamSourceOperator`/`StreamSinkOperator`/`WindowOperator` 各自的快照/恢复覆盖
- 测试: `TestOperatorSnapshot` (7 tests)

**Phase 4 - CheckpointCoordinator 集成**
- 新增 `CheckpointBarrierTracker`（core）：跟踪单链 barrier 传播与 ACK
- 新增 `GraphModelCheckpointExecutor`（runtime）：将 Coordinator 与执行引擎对接
- `StreamExecutionEnvironment.enableCheckpointing(long interval)` 配置 API
- 定时 barrier 注入 → 快照 → ACK → 持久化 → 完成通知
- 测试: `TestCheckpointEndToEnd` (3 tests)

**Phase 5 - 恢复流程实现**
- `GraphModelCheckpointExecutor` 启动时检查可恢复 checkpoint
- 各算子 `restoreState()` 从快照反序列化恢复状态
- Source 恢复 offset（`CheckpointedSourceFunction`）、Sink rollback + beginTransaction（`TwoPhaseCommitSinkFunction`）
- 测试: `TestCheckpointRecovery` (9 tests)

**Phase 6 - E2E 测试与文档更新**
- 新增 5 个 E2E 测试类: `TestE2ESimplePipeline` (4), `TestE2ECheckpointAndRecovery` (4), `TestE2ETwoPhaseCommitSink` (3), `TestE2EWindowOperatorWithCheckpoint` (4), `TestE2EMultipleCheckpoints` (5)
- 更新 `checkpoint-design.md` §10：集成状态从"未对接"改为"已对接"
- 更新 `graph-model-design.md` §7+§9：反映图模型路径已对接（单链管线）
- 更新 `README.md`：状态标记和心智模型
