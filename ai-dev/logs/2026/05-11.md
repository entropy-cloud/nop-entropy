# 2026-05-11

## Plan 14: nop-job 质量优化 — completed

4 phases executed in parallel, all passed `mvn test -pl nop-job`.

### Phase 1 — TimeoutChecker N+1 修复
- `IJobFireStore.batchLoadFires()` + `IJobScheduleStore.batchLoadSchedules()` 批量 IN 查询
- `JobTimeoutCheckerImpl.scanOnce()` 改为两步预加载（fireIds → fires → scheduleIds → schedules）再逐个 tryMarkTimeout
- SQL 查询从 2N 降到最多 2

### Phase 2 — Planner 锁冲突可观测性
- 仿照 `IOrmMetrics`/`OrmMetricsImpl` 模式，创建 `IJobPlannerMetrics` 接口 + `JobPlannerMetricsImpl`（使用 `GlobalMeterRegistry.instance()`）+ `EmptyJobPlannerMetrics`
- `nop.job.planner.due-count` / `nop.job.planner.lock-conflict` 两个预创建 Micrometer Counter
- `JobPlannerScannerImpl` 注入 `IJobPlannerMetrics`（默认 `EmptyJobPlannerMetrics`），不直接依赖 `MeterRegistry`
- 锁冲突 > 0 时输出 DEBUG 日志含冲突 scheduleId 列表
- `app-engine.beans.xml` 中配置 `JobPlannerMetricsImpl` bean 注入

### Phase 3 — 内部错误消息规范化
- `JobCoreErrors` 新增 5 个 ErrorCode: `JOB_TIMEOUT`, `JOB_INVOKER_NOT_FOUND`, `JOB_CANCELED`, `JOB_OVERLAID`, `JOB_EXECUTION_FAILED`
- 使用原始字符串值作为 ErrorCode code（向后兼容已有存储数据和测试断言）
- 替换了 `JobTimeoutCheckerImpl`, `JobWorkerScannerImpl`, `JobFireStoreImpl`, `JobScheduleStoreImpl`, `DefaultJobExecutionContextBuilder` 中的所有硬编码字符串

### Phase 4 — 遗留代码清理
- `SimpleJobState` 和 `ScheduledJob` 已在之前重写中移除
- 发现 `TestLocalJobScheduler` 测试仍存在，`LocalJobScheduler` + `IJobScheduler` 标记为 `@Deprecated`（未删除）
