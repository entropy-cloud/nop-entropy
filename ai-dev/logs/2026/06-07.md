### 模拟 Shell 设计文档

- **新建 nop-ai-shell 模块设计**：`ai-dev/design/nop-ai-agent/nop-ai-shell-design.md`
  - 深度调研 nop-ai-toolkit（IToolExecutor, IToolExecuteContext, IToolFileSystem）和 nop-ai-shell（IShellCommand, ShellCommandExecutor, BashSyntaxParser）的源码
  - 识别接口差距：两套文件系统抽象（IResourceStore vs IToolFileSystem）、返回类型不匹配、完全未连接
  - 设计接口统一方案：ShellBashExecutor（IToolExecutor 实现）+ ShellExecutionAdapter（适配层）
  - 命令实现分层：P0（26 个基础文件命令）、P1（git/mvn/npm 委托模式）、P2（网络受限）
  - 变量展开设计：ShellExpander ($VAR, ${VAR}, $?)
  - 混合执行模式：VirtualCommand（纯 JVM）+ DelegatingCommand（真实进程）
  - 安全设计：命令白名单、路径安全、资源限制

- **新建 Shell 语法规范**：`ai-dev/design/nop-ai-agent/nop-ai-shell-syntax-spec.md`
  - Tier 1（必须支持）：简单命令、环境变量、管道、逻辑运算符、I/O 重定向、分组、子 shell、后台、注释
  - Tier 2（重要增强）：变量展开 $VAR/${VAR}/$?、通配符 *、Tilde ~
  - Tier 3（按需实现）：命令替换 $()、算术展开、控制流
  - 明确不支持：函数、brace expansion、process substitution、arrays
  - Parser 问题诊断：>& vs &> 匹配顺序、<<< 匹配、注释 # 支持
  - LLM 常见命令模式参考（按频率排列）
  - Exit code 约定和信号处理
