### Plan 282 执行：nop-wf 示例单元测试补充

- Plan: `ai-dev/plans/282-nop-wf-example-tests.md`
- **结果**：14 个测试方法全部通过，覆盖全部 15 个示例。`./mvnw test -pl nop-wf/nop-wf-service -am` 74 tests 0 failures。
- **执行中发现的额外缺陷（超出原 Phase 0 范围）**：
  1. **ref-action 引用 common action**：全部 15 个示例 + sub-workflow 的 `<ref-actions>` 引用了 oa.xwf 的 common action（agree/reject/confirm 等均为 `common="true"`），触发 `ERR_WF_STEP_REF_ACTION_IS_COMMON`，导致示例完全无法加载。修复：删除全部 `<ref-actions>` 块（common action 自动添加到所有步骤，步骤级 `<transition onAppStates>` 已足够）。
  2. **execGroup 步骤需要非 local action**：oa.xwf 的 agree/complete/confirm 均为 `local="true"`（步骤不退出），而 execGroup 通过步骤退出追踪完成。and-group/seq-group 的步骤调用 local agree 后不退出，组永远无法完成。修复：countersign/vote-sign/or-sign/sequential-approval/comprehensive-leave(vp-approval) 添加专用非 local action（review-agree/vote-agree/or-agree/seq-agree/vp-agree）。
  3. **due-time-expr 使用未定义变量**：timeout-auto/comprehensive-leave 的 due-time-expr 使用 `sysDate`，该变量在 wfRt eval scope 中不存在。修复：改用日期字符串 `'2099-12-31 23:59:59'`（ConvertHelper.toTimestamp 可解析）。
  4. **subprocess persist/transition 修复**：subprocess args（employeeId/handoverItems）缺 `persist="true"`，transition 条件无法读取 wfVars。flow 步骤 transition 含 `onAppStates="agree"` 但 sub-flow end 不设 appState。修复：添加 persist，移除 onAppStates。
  5. **nop-wf-core 编译产物过期**：Plan 281 的引擎修改（ExecGroupSupport/WorkflowEngineImpl/WorkflowStepImpl）未重新编译进 .jar，导致 execGroup 测试失败。修复：`./mvnw install -pl nop-wf/nop-wf-core -am -DskipTests`。
  6. **测试基础设施**：EnhancedMockWfActorResolver 需设置 IUserDelegateService stub（subprocess actor 检查触发 canDelegate NPE）。
