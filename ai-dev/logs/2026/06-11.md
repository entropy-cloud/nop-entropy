## Goal Driver 流程重构

**Scope**: `ai-dev/tools/opencode-goal-driver/`
**Design**: `ai-dev/design/opencode-goal-driver/goal-driver-flow-design.md`

### 改动内容

1. **`flows/goal-driver.json`**：重写顶层流程
   - 移除 `DETECT_START`（死代码，所有 transition 指向同一节点）
   - `HEALTH_CHECK` 从 tool step 改为 AI agent step，失败时支持 3 次无上下文重试
   - 新增 `PLAN_ROUTER` script 步骤，扫描活跃 plan 并路由到执行或 roadmap
   - 主流程中 `EXECUTE_PLAN` 和 `DEEP_AUDIT_LOOP` 改为子流程引用

2. **`flows/plan-execution.json`**（新建）：可复用的 plan 执行子流程
   - `EXECUTE` → `CLOSURE_VERIFY`（group: `SCRIPT_CHECK` + `AI_AUDIT`）→ `BUILD_VERIFY`
   - 各步骤失败时可回退重试

3. **`flows/deep-audit-loop.json`**（新建）：深度审计子流程
   - 自循环：`DEEP_AUDIT` → `DRAFT_PLAN` → `AUDIT_PLAN` → `EXECUTE_PLAN` → `DRAFT_PLAN`（下一个问题）→ ... → `DEEP_AUDIT`（再审计）
   - 直到 `DEEP_AUDIT` 返回 `clean`

4. **`src/flow-loader.js`**：
   - 移除 `detectStartPhase` 和 `execSync` 依赖
   - 新增 `planRouter` 脚本函数（同步文件扫描，无子进程）
   - `closureScriptCheck` 改为直接 import `inspectPlan()` 而非子进程

5. **`prompts/health-check.md`**（新建）：AI agent 可诊断修复编译/测试失败

6. **`ai-dev/design/opencode-goal-driver/`**：
   - 新建 `README.md`（目录索引 + 阅读顺序）
   - 新建 `goal-driver-flow-design.md`（流程设计说明 + Mermaid 图 + 决策理由）

### 关键决策

- `PLAN_ROUTER` 只返回 `execute` 或 `roadmap`，不区分 `audit` 分支 —— roadmap 完成后自然进入 audit
- `EXECUTE_PLAN` 的 `flowArgs` 传递 `PLAN_FILE`，确保子流程可访问当前 plan 路径
- `HEALTH_CHECK` 的 retry 使用 `retry`（无 `append`），保证每次重试是空上下文
- `SCRIPT_CHECK` 直接从 `check-plan-checklist.mjs` 动态 import `inspectPlan()`，不再 spawn 子进程

### 测试修复

3 个集成测试因 flow 结构变化（移除 `DETECT_START`/`FIX_BUILD`、入口改为 `HEALTH_CHECK`、引入子流程）而失败，已修复：

1. **"completes full dev loop + audit loop"**：注入 mock subflows，移除旧引用
2. **"handles execute entry via PLAN_ROUTER"**：替代 `DETECT_START` 测试，计数器防循环
3. **"retries HEALTH_CHECK on failure"**：替代 `FIX_BUILD` 测试

另发现 `DEEP_AUDIT_LOOP` transition 名 `"completed"` 与 engine 子流程返回的 `"complete"` 不匹配，已修正。

### 子流程 marker 透传修复

子流程结果原本丢失了内部最后一步的 marker（硬编码为 `"complete"`/`"failed"`），修复后实际 marker 被透传给父步骤：

1. **`_result(status, stepCount, marker)`**：新增 `marker` 参数，记录最后一步的实际 marker
2. **`_executeSubflowStep()`**：优先使用 `childResult.marker` 而非硬编码
3. **主循环**：`transition.done` 路径将 `result.marker` 传入 `_result()`
4. **级联更新** flow JSON 的 subflow 步骤 transitions 使用实际 marker 名（`"pass"`/`"clean"` 替代 `"complete"`），测试 mock 同步更新

54/54 测试通过。
