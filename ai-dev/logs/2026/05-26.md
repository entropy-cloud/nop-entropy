# 2026-05-26 nop-code Module Improvement

## Summary
Deep audit and P0/P1 bug fix cycle for nop-code module. 4 parallel explorers audited service layer, core, graph/flow, and language adapters. Found and fixed 9 bugs total.

## Fixes Committed

### Commit 1: P0 Bug Fixes (5 bugs)
- **CommunityDetector** (nop-code-graph): `getByQualifiedName(symbolId)` → `getById(symbolId)` — caused all community labels to show "unknown"/"cluster_N" instead of meaningful package-based labels
- **searchBySymbolName** (nop-code-service): Removed broken `FilterBeans.eq("kind", language)` — NopCodeSymbol has no language column, was filtering wrong field
- **getSymbolSourceCode/showSymbolSource** (nop-code-service): Implemented source code retrieval from stored file sourceCode in DB — was always returning null with TODO comment
- **ChangeAnalyzer** (nop-code-flow): Wrapped Process/BufferedReader in try-with-resources — was leaking file descriptors on every git diff call
- **FlowDetector** (nop-code-flow): Fixed Spring entry point detection to match class names (Controller/Endpoint/Listener) and extData instead of checking annotation FQNs in qualifiedName (which never matches)

### Commit 2: P1 Fixes (3 bugs)
- **findImplementations**: Fixed superTypeId/qualifiedName key mismatch in BFS by building id-to-qualifiedName lookup map; removed dead symbolMap code
- **searchFullText/searchCombined**: Added language post-filter via NopCodeFile.language since NopCodeSymbol has no language column
- Added filterByLanguage helper for consistent language filtering

### Commit 3: JSON Safety
- **JavaFileAnalyzer**: Replaced hand-rolled JSON string concatenation in buildExtData() and toJson() with JsonTool.stringify()

### Commit 4: Adversarial Review Fix
- **entityToCodeSymbol**: Added missing extData mapping — async/synchronized/native/volatile/transient flags were lost when loading symbols from DB

## Round 2: Design Goal Implementation

### TypeScript Call Graph Extraction
- Added walkNodeForCalls() to TypeScriptCodeFileAnalyzer
- Extracts call_expression nodes with member_expression support (obj.method pattern)
- Populates CodeMethodCall for TS files — call hierarchy now works for TypeScript

### Java RecordDeclaration Support
- Added visit(RecordDeclaration) to JavaFileAnalyzer.IndexVisitor
- Records indexed as CLASS kind with implemented types and annotations
- Previously records (JDK 16+) were silently dropped

### Python Docstring Extraction
- Added extractDocstring() helper to PythonCodeFileAnalyzer
- Extracts first expression_statement string as documentation for classes/functions
- Handles triple-quoted and single-quoted docstrings

### Semantic Edge Model Implementation
Core model (nop-code-core/semantic/):
- CodeSemanticEdge: source/target symbol IDs, relation type, confidence, rationale
- EdgeConfidence enum: EXTRACTED(10), INFERRED(20), AMBIGUOUS(30)
- SemanticRelationType: 8 predefined types (semantically_similar_to, etc.)
- ISemanticEdgeExtractor interface

Deterministic extractors (nop-code-graph/semantic/):
- NameSimilarityExtractor: Jaccard similarity on normalized symbol names (threshold 0.7)
- DocKeywordExtractor: keyword overlap from documentation (threshold 0.5)
- AnnotationPatternExtractor: shared annotation patterns

ORM + Generated stack:
- nop_code_semantic_edge table with indexes
- Full codegen: DAO entity, BizModel, xmeta, xbiz, web views
