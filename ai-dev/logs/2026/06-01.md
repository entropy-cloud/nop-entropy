## Hollow Implementation Scanner Tool

Created `ai-dev/tools/scan-hollow-implementations.mjs` — a heuristic scanner that detects hollow/incomplete implementations across the Java codebase.

### Detection Patterns
| ID | Pattern | Severity |
|----|---------|----------|
| P1 | `UnsupportedOperationException` in non-test code | high |
| P2a | Empty method bodies | high |
| P2b | Bare `return null;` as sole statement | medium |
| P3 | Empty catch blocks | high |
| P3b | catch blocks with only printStackTrace/LOG | medium |
| P4 | TODO/FIXME markers | low |
| P6 | Stub/Placeholder/Dummy naming in production code | medium |
| P6b | "not yet implemented" / "minimal implementation" comments | high |
| P8 | Bare `continue;` in loops (suspected silent skip) | medium |

### Full Repo Results
- **2792 total findings** (src/main only)
- **56 high severity**: 53 `UnsupportedOperationException`, 3 "not yet implemented" comments
- Top modules: nop-kernel (15), nop-format (15), nop-stream (9), nop-persistence (5), nop-code (4)
- 1725 `return null;` findings (medium, needs human judgment)
- 654 bare `continue;` findings (medium, needs human judgment)

### Usage
```bash
node ai-dev/tools/scan-hollow-implementations.mjs                    # full repo, summary
node ai-dev/tools/scan-hollow-implementations.mjs --module nop-stream # single module
node ai-dev/tools/scan-hollow-implementations.mjs --severity high     # high+ only
node ai-dev/tools/scan-hollow-implementations.mjs --format json       # JSON output
```

### Key Bug Fixed During Development
The severity filter was inverted (`>=` instead of `<=`) due to `SEVERITY_ORDER` being sorted most-severe-first (index 0 = critical). Fixed to `severityRank(f.severity) <= minSev`.
