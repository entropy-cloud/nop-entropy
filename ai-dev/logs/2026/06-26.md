### docs-for-ai 补全：nop-report 报表引擎使用指南

- 类型：docs-for-ai 文档补全（无代码变更；与 CodeRule/ChangeLog 同类缺口）
- 触发：用户指出"nop-report 可导出 pdf/xlsx/html 都可作打印载体，套打有支持，需在 docs-for-ai 整理使用指南"
- 调研（子代理 + 自读实测）：
  - 引擎入口 `IReportEngine`（getRenderer/getHtmlRenderer/generateFromXptModel/buildXptModelFromImpModel）；**无独立 ReportService 类**，应用在 BizModel 注入调用（范例 `ReportDemoBizModel`）
  - 模型 = XPT 工作簿（`.xpt.xlsx`/`.xpt.xml` → `ExcelWorkbook`），单元格行为三入口：文本 `*=^ds!field`/`${...}`、批注 `key=value`→`XptCellModel`、配置页 `XptWorkbookModel`
  - 渲染类型 html/xlsx/pdf（`XptConstants.RENDER_TYPE_*`）；**PDF 是 PDFBox 直接渲染（非 xlsx 转换）**，三渲染器共享展开逻辑
  - 套打机制确认：`ExcelImage.print=false`（OOXML `fPrintsWithSheet="0"`）背景图屏幕可见、打印隐藏；范例 `nop-report-demo/.../base/09-套打.xpt.xlsx`，测试 `TestReportDemoBizModel.testFormPrinting()`
  - 数据集：程序构造 `ReportDataSet` 或 `xpt-rt.xlib` 标签（UseJdbcDataSet/UseOrmDataSet/UseQueryDataSet）在 beforeExpand 执行
- 做了什么：
  - 新建 `docs-for-ai/03-runbooks/generate-report.md`：核心概念 / 引擎入口 / BizModel 调用范例 / XPT 单元格语法（文本+批注+XptCellModel 指令表）/ 数据集（程序+SQL 标签）/ 套打三步 / imp 模板自动生成 / VFS 路径 / 反模式
  - `docs-for-ai/03-modules/nop-report.md`：概览补三输出载体+套打+imp 自动生成；加能力入口表指向 runbook；实体表前注明"渲染入口是 IReportEngine 非 CRUD BizModel"；修正原"输出格式：XLSX、PDF、DOCX"（实际 html/xlsx/pdf）
  - `docs-for-ai/04-reference/source-anchors.md`：加 `REPORT-001`(引擎入口+常量) / `REPORT-002`(XptCellModel 指令+展开) / `REPORT-003`(套打 ExcelImage.print + 三渲染器)
  - `docs-for-ai/INDEX.md` + `docs-for-ai/03-runbooks/README.md`：加 generate-report 路由
- 关键决策：作为高频任务独立成 runbook（与 CodeRule/ChangeLog/interceptor 一致），nop-report.md 回归概览
