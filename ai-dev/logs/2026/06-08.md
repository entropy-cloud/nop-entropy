### 分支亲和调度设计文档

- 新建设计文档：`ai-dev/design/nop-ai-agent/nop-ai-agent-branch-affinity-scheduling.md`
  - 核心决策：**Git Branch 作为调度亲和单元**
  - 调度单元 = (project, branch)，同一分支同时只在一个节点的一个进程内操作
  - Agent 是分布式开发者——复用 git 的分布式协作模型
  - 双环境共存：Java + Node.js 工具链在同一 session 中共存
  - 子 Agent 按需隔离：默认共享工作目录，探索性任务可选 git worktree
  - 提交与合并：Agent 不自动 push 到主干，小步提交，合并需人工/Merge Agent 触发
  - **文件快照系统**：基于 JGit 的独立外部 git 方案（opencode/cline 验证），tree hash 无 commit，零侵入
  - **Snapshot vs Worktree 关系**：Snapshot 解决 undo/回滚（Layer 2 必须），Worktree 解决并行隔离（Layer 4 可选）
  - Worktree 详细设计：外部存储（`{data_dir}/ai-worktrees/`）、命名分支（`-B`）、Symlink 大目录、Fast Resume、Slug 验证、Orphaned 回收
  - 拒绝了 8 项：分布式文件系统协调、Session 级调度、跨节点 Worktree、Agent 自动 Merge、项目 git 做快照、项目内 Worktree、Detached HEAD、依赖系统 git CLI
  - 对分层架构的影响：Layer 2 新增 `IFileSnapshotService`，Layer 3 新增 `IBranchAffinity`，Layer 4 新增 Worktree 生命周期管理器
  - 数据模型：Affinity Table + Worktree Registry（扩充字段）+ Snapshot Record（嵌入 ai_session_message）

#### 工作循环防护（query.py, 1057行）

- `max_turns` 双默认值：`run_query`=200, `QueryEngine`=8
- `MAX_SAFE_COMPLETION_TOKENS=128K` — per-request output token cap
- 17 种 prompt-too-long 错误模式检测
- 三级恢复策略：completion limit 适配 → reactive compact（force=True）→ 终止
- 空 assistant 消息检测 + 终止
- `return_exceptions=True` — 多工具并发执行异常隔离
- Tool output offloading — 大输出写文件，返回 preview

#### 上下文压缩（compact/__init__.py, 1725行）

- 四级压缩瀑布：microcompact → context collapse → session memory → full LLM summarization
- Auto-compact 阈值：`context_window - 20K(reserved) - 13K(buffer)` ≈ 167K for 200K context
- `MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES=3` — 连续失败保护
- Tool pair 保护 — 压缩边界不切断 tool_use/result 对
- Reactive compact — API 返回 prompt too long 时紧急压缩
- PTL retry — 压缩请求本身 prompt too long 时截断旧消息重试（最多 3 次）
- 8 类 `CompactAttachment` — 压缩时保留 focus/artifacts/skills 等状态

#### 权限检查（permissions/checker.py, 200行）

- 硬编码敏感路径保护（`.ssh`/`.aws`/`.gcloud`/`.azure`/`.gnupg`/`.docker`/`.kube`）— 不可覆盖
- 三级权限模式：`default`（读写分离确认）/ `plan`（只读）/ `full_auto`
- 用户自定义 path rules（glob）+ command deny patterns
- Bash 安装命令友好提示

#### 运行时 Focus 追踪（query.py）

- 9 类 tool_metadata 追踪器（read_file_state, invoked_skills, async_agent_state, recent_work_log 等）
- `_append_capped_unique()` — 新值推入、重复移到末尾、超限截断
- 压缩时通过 CompactAttachment 注入到新对话

- `call-agent` 支持 `maxTurns` 参数，默认 20
- Layer 2 实现 microcompact + context collapse，Layer 3 可选 LLM summarization
- 内置敏感路径保护，不可被 Delta 定制覆盖
- `ai_session_context` 表存储 focus 追踪数据
