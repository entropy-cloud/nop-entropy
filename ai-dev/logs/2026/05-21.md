# 2026-05-21

## nop-message-pulsar 实现完成

**Plan**: `ai-dev/plans/31-nop-message-pulsar-implementation-plan.md`

完成 nop-message-pulsar 从代码骨架到最小可用 Pulsar 消息服务实现的全部 5 个阶段：

### Phase 1 - PulsarHelper 补全
- `buildApiMessage`: 从 Pulsar Message 构建 ApiRequest（提取 key/value/messageId/topicName/eventTime/properties/publishTime/sequenceId）
- `encodeValue`: String 原样返回，Number/Boolean 调用 toString()，其他类型 JSON.stringify()，null 返回 null
- `_buildPulsarMessage` 中 encodeValue 返回 null 时跳过 property 设置

### Phase 2 - PulsarMessageService 生命周期与 IoC 集成
- pom.xml 依赖从 `pulsar-client-api` 改为 `pulsar-client-original`
- `nop-dependencies/pom.xml` 增加 `pulsar-client-original` 版本管理
- PulsarProducerConfig 增加 batchingEnabled/batchMaxMessages/sendTimeout + @DataBean
- PulsarConsumerConfig 增加 ackTimeout/negativeAckRedeliveryDelay/maxTotalReceiverQueueSizeAcrossPartitions + @DataBean
- PulsarMessageService 增加 init()/destroy() 生命周期方法，init() 创建 PulsarClient + defaultProducer，destroy() 按序关闭
- 删除死代码 subscriptionConfigs 字段
- topicSchemas 初始化为 ConcurrentHashMap
- doSubscribe 传递 topic/subscriptionName/subscriptionType 到 ConsumerBuilder
- 创建 autoconfig 链路: nop-message-pulsar.beans → pulsar-defaults.beans.xml
- 新增 PulsarErrors 错误码定义

### Phase 3 - PulsarConsumeTask 消费循环修复
- start() 在 runTask() 前设置 active = true
- Executor 改为 ExecutorService
- runTask 增加消费失败退避策略（默认 1 秒，可配置）
- seekToPosition() 添加 TODO 注释
- PulsarMessageSubscription.doOnCancel 按 task.stop() → consumer.close() → executor.shutdown() 顺序关闭

### Phase 4 - nop-message-core Bug 修复
- `LocalMessageService.invokeMessageListener` 异步路径传递 `r`（解析后的值）而非 `ret`（CompletionStage 对象）
- 为每个消费者的 onMessage 调用添加 try-catch，单个消费者异常不阻断后续消费者
- `MultiMessageSubscription` cancel/suspend/resume 使用 addSuppressed 保留所有异常

### Phase 5 - 测试
- TestPulsarHelper: buildApiMessage 字段映射 + encodeValue 各类型测试（7 个测试用例）
- TestPulsarConsumeTask: start/stop 状态验证（2 个测试用例）
- TestPulsarIoC: 配置注入 + fail-fast 验证（4 个测试用例）
- TestLocalMessageServiceAsync: CompletionStage<Acknowledge> 正确处理 + 异常隔离（2 个测试用例）
