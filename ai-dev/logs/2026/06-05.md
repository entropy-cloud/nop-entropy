# 2026-06-05 Development Log

## Plan 116: nop-code Data Model Enhancement — Provenance, Heuristic Edges, Framework Extraction

**Status**: In progress (Phase 1-3 completed, Phase 4 documentation in progress)

### Changes Made

#### Phase 1: Edge Provenance and Metadata
- Added `PROVENANCE` VARCHAR(20) column to all 5 edge tables in ORM (`nop_code_call`, `nop_code_usage`, `nop_code_inheritance`, `nop_code_annotation_usage`, `nop_code_semantic_edge`)
- Added `METADATA` JSON column to `nop_code_call` and `nop_code_usage` (most complex edge types)
- Created `EdgeProvenance` enum: `AST_EXTRACTION`, `SYMBOL_SOLVER`, `HEURISTIC`, `FRAMEWORK_INFERENCE`, `MANUAL`
- Updated all core models (`CodeMethodCall`, `CodeInheritance`, `CodeAnnotationUsage`, `CodeSemanticEdge`) with `provenance` field
- Updated `CodeMethodCall` with `metadata` field
- Updated all language adapters (Java, Python, TypeScript) to set provenance on extracted edges
- Updated `CodeIndexService` persistence to map provenance and metadata

#### Phase 2A: Symbol Model Enhancement
- Added `FILE_PATH` and `LANGUAGE` denormalized columns to `nop_code_symbol` table
- Added `MODIFIER_EXPORTED = 1 << 9` to `CodeSymbol`
- Added `ROUTE` (value=100) to `CodeSymbolKind` enum and `code/symbol_kind` dictionary
- Added `TYPE_OF` (value=100) and `INSTANTIATES` (value=110) to `CodeUsageKind` and `code/reference_kind` dictionary
- Updated `CodeIndexService` to populate denormalized `filePath`/`language` on symbol persistence
- Updated `CodeSymbolConverter`, `SymbolDTO` with new fields
- Java adapter sets `EXPORTED` flag on public/protected classes and methods

#### Phase 2B: Spring Framework Route Extraction
- Created `CodeRouteInfo` model class (httpMethod, routePath, handlerSymbolId, handlerQualifiedName)
- Added `routes` list to `CodeFileAnalysisResult`
- Implemented Spring route extraction in `JavaFileAnalyzer`:
  - Detects `@RequestMapping` (class-level prefix) + `@GetMapping`/`@PostMapping`/`@PutMapping`/`@DeleteMapping`/`@PatchMapping`
  - Merges class prefix + method path
  - Stores route info in handler method's `extData` JSON
  - Creates `CodeRouteInfo` objects in analysis result

#### Phase 3: Heuristic Edge Synthesis Engine
- Created `IHeuristicEdgeSynthesizer` interface and `HeuristicContext` data class in `nop-code-core`
- Implemented `InterfaceImplSynthesizer`: synthesizes edges from interface method calls to all known implementations
- Implemented `SpringEventSynthesizer`: matches `@EventListener` methods with `publishEvent()` call sites (first iteration: limited precision)
- Integrated into `CodeIndexService`: after persisting AST edges, builds `HeuristicContext` and runs all registered synthesizers
- Deduplication: queries existing `(indexId, callerId, calleeId)` edges before inserting synthesized edges
- Synthesized edges use `line=-1, column=0` sentinel values
