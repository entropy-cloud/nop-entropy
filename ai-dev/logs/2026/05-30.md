## nop-stream Plan 75 Post-Closure Fix: AR-19 Checkpoint Ghost Regression

**Plan**: ai-dev/plans/75-nop-stream-round9-and-p2-audit-remediation.md (post-closure fix)

**Root Cause**: Plan 75 Phase 3 AR-19 ghost checkpoint fix removed `completableFuture.complete()` from `PendingCheckpoint.acknowledgeTask()`, but `CheckpointCoordinator` relied on a `whenComplete` callback (triggered by that completion) to drive `completePendingCheckpoint()`. This created a deadlock: the future was never completed because the completion trigger was removed, but the code that would complete it only ran when the future completed.

**Fix (commit c89f3e3dc)**:
- `PendingCheckpoint.acknowledgeTask()`: future not auto-completed (keeps AR-19 guarantee)
- `PendingCheckpoint.forceComplete()`: new public method for explicit completion after storage
- `CheckpointCoordinator.acknowledgeTask()`: directly checks `isFullyAcknowledged()` and calls `completePendingCheckpoint()` instead of relying on `whenComplete` callback
- `CheckpointCoordinator.completePendingCheckpoint()`: calls `pending.forceComplete()` after successful storage
- `CheckpointCoordinator.shutdown()`: resets `numPendingCheckpoints` to 0
- Test alignment: `TestRemoteDataExchange` expects `StreamException` (Phase 5 error code migration)
- Test alignment: standalone `PendingCheckpoint` tests use `forceComplete()`

**Tests**: All 1399 tests pass (core 822 + cep 216 + runtime 322 + connector 39)
