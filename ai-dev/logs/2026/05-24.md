# 2026-05-24

## Plan 47: nop-stream 分布式执行路径接线

**Status: completed**

### What was done

Wired the distributed execution path so `StreamExecutionEnvironment.execute()` can choose LOCAL or DISTRIBUTED mode.

#### Phase 1: Direct 路径并行度修复 + SPI 接口 (nop-stream-core)
- New `DeploymentMode` enum (LOCAL, DISTRIBUTED)
- New `IStreamExecutionDispatcher` SPI interface
- New `SubtaskTask` class (per-subtask execution with lifecycle management)
- Fixed `execute()` direct path: replaced `Task(vertex, 0)` with per-subtask `SubtaskTask` iteration
- Added distributed branch in `execute()`: delegates to `IStreamExecutionDispatcher` when DISTRIBUTED
- Tests: 5 new tests in `TestSubtaskExecution`

#### Phase 2: 控制面接口定义与 InMemoryClusterRegistry (nop-stream-runtime)
- New `IStreamTaskRpcService` interface (receiveAssignment, triggerCheckpoint, cancelTask)
- New `IStreamCoordinatorRpcService` interface (receiveCheckpointAck)
- `TaskManager` implements `IStreamTaskRpcService` directly (no adapter)
- New `InMemoryClusterRegistry` (ConcurrentHashMap-backed, for embedded mode)
- Tests: 5 new tests in `TestInMemoryClusterRegistry`

#### Phase 3: JobCoordinator 控制面迁移 (nop-stream-runtime)
- `JobCoordinator` constructor accepts `Map<String, IStreamTaskRpcService>`
- `JobCoordinator` implements `IStreamCoordinatorRpcService`
- `assignTasks()` and `triggerCheckpoint()` use strong-typed RPC services when available
- `TaskManager` holds `IStreamCoordinatorRpcService` reference for ACK path
- Fallback to IMessageService when no RPC services configured (backward compat)
- All 11 existing JobCoordinator tests pass

#### Phase 4: 嵌入式分布式编排器 (nop-stream-runtime)
- New `EmbeddedDistributedExecutor` implements `IStreamExecutionDispatcher`
- Creates N TaskManagers, registers them as `IStreamTaskRpcService`
- Uses `RemoteGraphExecutionPlanBuilder` for remote transport
- Direct `installInvokable()` calls (orchestration plane, no serialization)
- E2E test: `testDistributed_sourceMapSink` with parallelism=2, DISTRIBUTED mode

### Architecture decision

Three-plane communication model for embedded distributed mode:
- **Control plane**: `IStreamTaskRpcService` / `IStreamCoordinatorRpcService` (强类型接口，直接方法调用)
- **Data plane**: `IMessageService` + RemoteResultPartition/RemoteInputChannel
- **Orchestration plane**: Direct Java calls (installInvokable, no serialization)

User feedback: Removed adapter pattern (StreamTaskRpcServiceAdapter/StreamCoordinatorRpcServiceAdapter) in favor of direct strong-typed interface usage. TaskManager IS-A IStreamTaskRpcService; distributed mode gets remote proxies from Nop RPC framework.
