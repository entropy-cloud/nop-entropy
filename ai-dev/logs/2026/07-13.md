#### 关键学习

1. **minified js grep 不可靠**：plan 289 用 minified js grep 做调查，得出"Flux 不支持 aside"等错误结论。实际 minified js 中字符串可能被压缩成变量引用（`initApi`/`submitOnChange` 等肯定存在的属性也是 0 次出现）。**正确方法**：直接核对 Flux TypeScript 源码定义（`packages/*/src/schemas.ts` 和 `packages/flux-core/src/types/`）。
2. **Flux 属性命名无统一规律**：之前 plan 289 推断"AMIS `*On` 后缀统一改为 `*When`"是错误的。实际：`visibleOn/disabledOn/requiredOn/readonlyOn` 是删除 On 后缀；`clearValueOnHidden` 是结构重组为嵌套对象；`asideResizor` 是改为形容词形式 `asideResizable`；`initFetch` 改为 `autoInit`。必须基于具体属性核对，不能泛化。
3. **TypeScript d.ts 与运行时 minified js 的差异**：flux-guide/flux-types/schema.d.ts 是手工抽取的子集，可能与实际 packages/*/src/schemas.ts 不一致。本次发现 flux-types/schema.d.ts 的 PageSchema 只列了 aside/asidePosition，但实际 packages/flux-renderers-basic/src/schemas.ts 有完整 6 个 aside 属性。**权威源是 packages/*/src/schemas.ts**。

#### 第二次更新：澄清 `@cfg:` 语法 + 集中化 value resolver 文档

- 调研结论：YAML/JSON DSL 加载阶段的 `@`-prefix value resolver 注册于 `ValueResolverCompilerRegistry.DEFAULT`（6 个内置：`@cfg:` / `@i18n:` / `@var:` / `@uuid:` / `@load:` / `@empty:`），`IValueResolverCompiler.BIND_EXPR_SYMBOL = '@'`，`ConfigValueResolver` 类注释明确语法为 `@cfg:a.b.c,b.c.e|3`（**不是** `${cfg:xxx}`）
- 关键区分：XDSL delta 加载（`JsonTool.loadDeltaBeanFromResource` → `DeltaJsonOptions`）与 IoC 容器（`ConfigExpressionProcessor` → `IBeanPropValueResolver`）是**两套并行的 `@cfg:` 实现**，写法相同但实现独立。IoC 额外支持 `@r-cfg:`（reactive）/ `@bean:` / `@inject-ref:` 等；YAML 加载只有 6 个通用 resolver，加载期一次性求值（非 reactive）
- 新建 `docs-for-ai/02-core-guides/value-resolvers.md` 集中说明 `@`-prefix value resolver 通用机制（语法表、内置 resolver、生效入口、与 IoC 对比、扩展点）
