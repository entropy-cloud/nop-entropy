## plan 289 flux-web.xlib Layout 语法完整性修复 — 执行完成

**Plan**: [`289-flux-web-layout-syntax-completeness.md`](../plans/289-flux-web-layout-syntax-completeness.md)
**Status**: Plan 全部 3 个 Phase 完成，待 closure audit

### 摘要

修复 plan 287（首版 Flux xlib）遗留的 layout 语法支持缺陷。审计发现 `flux-web.xlib:GenFormSimpleCell` 中 mandatory 处理缺少 `formCell.mandatory ||` 前缀，导致 layout `*` 必填修饰符（`layout-syntax-reference.md` §3.1）和查询必填（§11.4）在 Flux 渲染器下完全失效。

### 执行结果

**Phase 1**：修复 mandatory 优先级链
- `flux-web.xlib:338-340` 补 `formCell.mandatory ||` 前缀，与 `web.xlib:369-371` 完全一致
- 新增 fixture：`test-flux-form-mandatory.{view.xml,xmeta}` + 4 个 page.yaml wrapper
- 在 `TestFluxWebGen.java` 新增 4 个 `@Test` 方法（3 个验证 + 1 个对照）

**Phase 2**：补全 cell 动态属性 + 裁定 DefaultAttrs 缺失
- **重要发现**：调查 Flux 运行时（解压 `pkg-nop-chaos-flux-C3hPTSdm.js.gz`）发现 Flux 命名规律——AMIS `*On` 后缀在 Flux 中改为 `*When`：
  - AMIS `requiredOn` → Flux `requiredWhen`（13 次出现）
  - AMIS `clearValueOnHidden` → Flux `clearValueWhenHidden`（11 次出现）
- `GenFormSimpleCell` cell 对象新增 `requiredWhen: cellModel?.requiredOn` 和 `clearValueWhenHidden: cellModel?.clearValueOnHidden`
- `FluxFormDefaultAttrs` 4 个缺失属性（`initFetch`/`silentPolling`/`wrapWithPanel`/`canAccessSuperData`）经调查全部 0 次出现，裁定不补全
- `FluxPageDefaultAttrs` 5 个 `aside*` 属性基于 analysis §2.2 确认 Flux 无 aside 容器，不补全
- 新增 3 个测试（2 个 form 级 + 1 个 page 级对照）

**Phase 3**：文档同步
- `layout-syntax-reference.md` 顶部声明改为同时适用于 AMIS 和 Flux
- `flux-rendering.md` 新增"Flux 已确认不支持的 form/page 属性"表 + cell 命名映射说明
- `source-anchors.md` `EXT-009` 描述补充

### 关键学习

1. **plan closure audit 的颗粒度问题**：plan 287 closure audit 通过了，但本次审计发现其对比粒度停留在"标签名集合一致 + 控件 type 名映射正确"，未对 layout 语法核心标签 `GenFormSimpleCell` 逐行比对。导致 `mandatory` 优先级链缺失这种关键 bug 漏过 closure audit。
2. **Flux 命名规律**：通过解压 minified js 并 grep 发现 Flux 把 AMIS 的 `*On` 后缀统一改为 `*When`。这是一个之前文档未记录的命名映射规律。
3. **view.xml cell 的 requiredOn 是子元素而非属性**：执行时发现 `<cell requiredOn="...">` 会报错 `attr-not-allowed`，因为 `requiredOn` 在 `disp.xdef:61` 中是子元素而非属性。修正为 `<cell><requiredOn>...</requiredOn></cell>`。
