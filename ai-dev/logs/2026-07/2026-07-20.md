## Changes

### Render mode switch (x:post-extends + c:print)

- **Problem solved**: `x:post-extends` cannot access tag local variables (`${view}`, `${page}`) — solved via `c:print` which preserves `${...}` as literal text
- **Key implementation details**:
  - `impl_flux_mode.xpl` output must be wrapped in `<lib>` root node — `genCpExtends` strips `<_>` dummy and validates children as XDSL sources against root xdef
  - `<source>` body should NOT be wrapped in `c:print` — outer `c:print` already protects during post-extends; tag invocation needs normal compilation
  - `xpl:is="c:print"` ignores wrapper element — use direct `<c:print>` instead
