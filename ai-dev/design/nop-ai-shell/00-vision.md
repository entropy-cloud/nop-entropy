**日期**：2026-06-08
**状态**：active

## 一、产品定位

nop-ai-shell 是一个**嵌入式类 Bash 命令执行框架**，服务于 AI Agent 的工具调用场景。它不是通用终端模拟器，而是一个嵌入式的命令执行引擎——AI Agent 构造命令行字符串，shell 解析并执行，将结构化结果返回给 Agent。

内置命令在 JVM 内纯 Java 实现，对于未注册的命令可通过**外部 shell 回退**（`nop-shell` 的 `ProcessBuilder`）委托给操作系统执行。

## 二、核心场景

1. **Agent 工具调用**：AI 构造 `ls -l /data` 或 `grep "error" < log.txt`，获得文本结果
2. **命令管道**：`cat file.txt | grep "pattern" | wc -l`，多阶段数据处理
3. **脚本编排**：`cd /workspace && ls && cat README.md`，多步操作串联
4. **后台任务**：长时间运行的命令异步执行，立即返回控制权
5. **混合执行**：内置 `echo` + 外部 `git` + 内置 `grep` 在同一管道中协作

## 三、设计约束

1. **内置命令纯 Java**：注册在 `ShellCommandRegistry` 中的命令不启动外部进程
2. **外部回退可选**：通过 `IShellRunner` 注入启用，未配置时未注册命令返回 exit code 127
3. **非交互式**：无终端、无 readline、无 tab 补全、无颜色
4. **内存管道**：命令间的管道是内存中的数据流，不通过 OS pipe
5. **异步优先**：管道各阶段并发执行，数据逐条流过，不支持一次性缓冲全部输出后再传递
6. **文本为主**：命令输入输出以文本为核心数据类型，通过扩展点支持二进制
7. **安全验证**：执行前通过命令校验器（`ICommandChecker`）预检整棵 AST，拒绝不合规命令

## 四、Non-Goals

- **不替代真正的 Bash**：不支持信号处理、进程管理、文件描述符继承
- **不做终端 UI**：不支持 TTY、颜色、光标控制
- **不追求 Bash 兼容性**：只支持 AI Agent 常用的语法子集
- **不做安全沙箱**：命令访问控制由上层（Agent 权限系统）负责
- **不在内部命令中调用 ProcessBuilder**：外部进程只在回退路径中出现

## 五、成功标准

1. 管道 `A | B | C` 三阶段真正并发流式执行，不是串行批处理
2. 后台命令 `cmd &` 立即返回，不阻塞调用方
3. 内置命令（`cd`、`ls`、`echo`、`pwd`、`grep`、`cat`、`wc`）行为与 Bash 一致
4. 管道中传递大数据（万行级别）不会 OOM，有自然的背压机制
5. 未注册命令在开启回退时可正确委托给 OS shell 执行并收集输出
6. 危险命令（`rm -rf /`、`:(){:|:&};:`）被验证器拒绝，不传递给 OS shell
